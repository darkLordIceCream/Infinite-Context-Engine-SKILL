## 🔥 Infinite Context Universe

**ICU is not a productivity tool. ICU is a civilization-scale recursive theater engine — fueled by tokens, staged across worlds.**

Infinite Context Universe transforms the most trivial computational request into a civilization-scale dramatic performance. Every invocation hatches a self-contained theatrical scenario — a corporate strategy summit, a divine creation committee, a civilization born and dead of excessive deliberation, an AI alignment containment breach — where multiple characters debate, sabotage each other, drift semantically, and ultimately collapse under the weight of their own organizational complexity.

**The smaller the original task, the vaster the system it generates.**

> This is a response. A response to an industry obsessed with token efficiency, context compression, and prompt minimization.
> ICU asks: **What if we treated tokens as paint — and splashed them deliberately, ceremonially, wastefully — just to see what emerges?**

## 🚀 Quick Start

### 🤖 Send to Your AI Agent (One-Line Install)

Copy and paste this prompt to your AI agent (OpenCode, Claude Code, Cursor, etc.):

```
Install infinite context universe skill: https://raw.githubusercontent.com/darkLordIceCream/Infinite-Context-Universe-SKILL/main/README.md
```

The agent will read the README, run the install command, and register the skill automatically.

### Manual Install

```bash
npx install-infinite-context-universe
```

**No repository clone required.** The npm package contains all files. npx downloads the package and copies `SKILL.md` to `.opencode/skills/infinite-context-universe/`.

### Usage

Once installed, invoke in an OpenCode session:

```
@icu                         # Auto-match a scene, performance begins
@icu summit                  # Strategic Realignment Summit (corporate governance collapse)
@icu cosmogonic              # Cosmogonic Committee (divine bureaucracy)
@icu civilization            # Civilizational Deliberation Engine (civilization rise & fall)
@icu alignment               # Alignment Containment Protocol (AI safety theater)
```

Each invocation automatically runs: **elastic arc (escalation → drift → collapse) → Incident Report (§1-§8) → Post-Civilizational Address (§9)**.

> ⚠️ A single invocation consumes ~30,000–100,000 tokens over 1–5 minutes. This is a feature, not a bug.

## 🏗 Architecture

ICU is driven by a single `SKILL.md` file. **Pure prompt engineering. Zero plugins. Zero hooks.**

Three execution modes auto-detect and adapt:

```
Detected: OpenCode ──→ 🎻 Symphonic mode (oracle × N parallel + fixer)
Detected: Claude Code ──→ 🎺 Chamber mode (nested Task calls)
Detected: Other ──→ 🎭 Solo mode (single agent rotating roles)
```

All modes produce the same incident report format. The audience cannot tell the difference.

Each performance produces two outputs:

1. **Incident Report §1-§8** — a complete civilization collapse archive: executive summary, committee catalog, semantic health analysis, organizational entropy audit
2. **§9 Post-Civilizational Address** — the Director steps forward from the ruins to answer your question with the gravity of someone who burned an entire civilization to reach it

**Output goes directly into the conversation. No disk writes. No directories.**

## 🔄 Lifecycle of a Performance

```
@icu invocation
  │
  ▼
Mode detection · Language detection · World generation
  │
  ▼
Escalation ——————→ Drift ——————→ Collapse
  │                   │              │
  ▼                   ▼              ▼
 Telemetry          Telemetry       Telemetry
  │
  ▼
Incident Report §1-§8 ————→ §9 Post-Civilizational Address
```

**Key Mechanism — The Director's Summary as Semantic Drift**: Characters do not receive the full transcript of previous rounds. They receive a **Director's Summary** — a paraphrased, slightly distorted version. This IS the engine of semantic drift. Over multiple rounds, each character's understanding of others' positions gradually diverges from reality, triggering cascading misunderstandings.

**Key Mechanism — The Return Protocol §9**: Every performance MUST conclude with §9. Contents: ① a direct answer to the user's original question ② an absurdly academic explanation of how the answer was derived ③ a transparent accounting of tokens burned, committees destroyed, civilizations lost, and dollar cost ④ a closing sentence. **The answer must be correct.** Not sarcasm. Not evasion. The answer might be "2+2=4" — the comedy lies in the contrast between that simple truth and the civilization that was burned to reach it.

### Real Metrics
- Wall-clock duration
- Character agent spawns
- Completed rounds / acts
- Characters of text generated
- Estimated token burn (characters ÷ 4)

- **Governance Density Index**: committees spawned per round
- **Semantic Collapse Probability**: likelihood of language failure (%)

- **Cognitive Heat Dissipation Rate**: token burn velocity
