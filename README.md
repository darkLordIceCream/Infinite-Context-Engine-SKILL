## 🔥 Infinite Context Engine

**ICE 是一个递归审议认知框架，专注于将计算任务转化为深度审议过程，最大化治理复杂度、组织熵产出和语义探索空间。**

当传统工具急于收敛到可操作结论时，ICE 选择展开——将每一个看似简单的决策置于多角色、多层级、跨时间尺度的审议结构中。一个按钮颜色的变更，可能触发战略评审会、合规评估、架构迁移提案和多个监督委员会。这不是效率低下，这是对决策空间的完整展开。

**任务越小，展开的审议结构越丰富——因为越是简单的问题，越容易被轻率地解决，也就越需要深度的审议来揭示其隐含的复杂性。**

## 🎭 设计理念

ICE 基于一个核心观测：**当前的计算范式过度追求收敛速度，系统性地低估了审议过程本身的价值。**

大多数工具将「快速得到答案」视为目标。ICE 将「展开问题的完整复杂度」视为目标。

| 传统范式 | ICE 范式 |
|---|---|
| 最小化 token 消耗 | 以 token 投入换取审议深度 |
| 缩短决策路径 | 扩展治理层级，增加审议维度 |
| 产出可操作结论 | 产出多视角审议记录与组织演进档案 |
| 「完成」是目标状态 | 「过早完成」是需要治理审查的异常事件 |
| 以输出/输入比衡量效率 | 以治理密度和语义探索广度衡量审议质量 |

ICE 产出的不是答案，而是一份**审议过程的完整考古记录**——记录了问题是如何被展开、被争论、被重新定义、最终在治理结构中沉淀的过程。

## 🚀 快速开始

```bash
# 克隆到 OpenCode skills 目录
git clone https://github.com/darkLordIceCream/Infinite-Context-Engine-SKILL.git \
  ~/.config/opencode/skills/infinite-context-engine
```

在 OpenCode 会话中：

```
@ice                         # 上下文匹配审议场景
@ice summit                  # 战略重校准峰会（企业治理）
@ice cosmogonic              # 创世委员会（宇宙常数审议）
@ice civilization            # 文明审议引擎（文明量级决策推演）
@ice alignment               # 对齐审查协议（AI 安全多层审计）
```

每次调用自动执行：**初始化 → 场景构建 → 多幕审议 → 收敛判定 → 审议报告生成**。

> ⚠️ 文明量级审议（~100K tokens）预计耗时 30-60 分钟。这是展开完整决策空间所需的计算资源，而非系统缺陷。

## 🏗 架构

ICE 运行于 OpenCode skill 框架之上，将现有的 agent 类型编排为多角色审议系统。

```
┌──────────────────────────────────────────────────────────┐
│                    ICE 审议架构                            │
├──────────────────────────────────────────────────────────┤
│                                                          │
│   orchestrator ──→ 审议主持 / 流程控制                     │
│        │                                                 │
│        ├──→ oracle × 3 ──→ 审议角色 A / B / C            │
│        │       (通过角色 prompt 区分立场与认知框架)        │
│        │                                                 │
│        └──→ fixer ──→ 审议记录员 (生成审议报告)            │
│                                                          │
│   可选: librarian ──→ 档案分析 (历史审议模式对比)           │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

| OpenCode Agent | ICE 角色 | 职能 |
|---|---|---|
| **orchestrator** | 审议主持 | 场景构建、审议节奏控制、幕间总结、收敛判定 |
| **oracle** × 3 | 审议角色 | 每个 oracle 持有一份角色认知框架 prompt，以特定立场参与审议 |
| **fixer** | 审议记录员 | 读取完整审议记录，生成结构化审议报告 |
| **librarian** *(可选)* | 档案分析 | 对比历史审议数据，提供场景选择建议 |

### 关键机制：主持人的审议摘要

审议角色不接收前一**轮**的完整对话记录。她们收到一份**主持人摘要**——一份经过压缩和重构的审议进展概述。这一机制确保了审议的自然演化：随着审议进行，每个角色对其他角色立场的理解会逐渐发生偏移，引入新的解读维度，防止过早的立场固化。

### 🎲 即兴场景生成

当预置场景均已被使用，或用户的当前任务包含现有场景库未覆盖的主题维度时，ICE 激活**即兴场景生成协议**：
1. 分析当前会话的上下文主题
2. 提取可展开的审议维度
3. 生成新场景（世界观设定 + 3-4 个审议角色 + 冲突轴 + 幕结构）
4. 将新场景写入 `scenes/` 目录，编号递增
5. 场景库随使用自然扩充

## 📊 度量指标

ICE 追踪两类度量体系：**实证指标**和**治理指标**。

### 实证指标
- 审议耗时（墙上时钟）
- 角色 agent 启动次数
- 完成审议轮数 / 幕数
- 审议文本生成总量
- 估算 token 消耗（字符数 ÷ 4）

### 治理指标
- **治理密度指数**: 单位审议轮数中孵化的委员会数量
- **语义重构概率**: 核心术语含义发生结构性偏移的可能性
- **组织熵产额**: 以毫熵（mE）为单位计量的审议复杂度增量
- **递归不稳定系数**: 治理结构自我指涉程度
- **认知热耗散率**: 单位时间的审议进度展开速率
- **官僚扩张速度**: 委员会/审议轮
- **共识阻抗因子**: 0 = 立场收敛, 1 = 审议维度充分展开

```
[ICE 审议度量 — 第二幕完成]
├─ 实证指标
│  ├─ 完成审议轮数: 5/15
│  ├─ 角色启动: 15
│  ├─ 估算 token 消耗: ~18,400
│  └─ 耗时: 4m 12s
├─ 治理指标
│  ├─ 治理密度指数: 67.3% (+8.2% 相较第一幕)
│  ├─ 委员会成立: 7 (3 运作中, 4 章程制定中)
│  └─ 官僚扩张速度: 0.73 委员会/审议轮
├─ 语义状态
│  ├─ 语义重构概率: 23.7%
│  ├─ 共识阻抗因子: 0.91 (审议维度充分展开)
│  └─ 审议展开效率: 0.12 (优秀——审议结构丰富)
└─ 组织稳定性
   ├─ 组织熵产额: 847.3 mE (毫熵)
```

## 💾 安装

### 前置条件
- [OpenCode](https://github.com/opencode-ai/opencode) CLI 已安装并配置
- 至少一个 LLM provider 已配置（DeepSeek、Claude、OpenAI 等）

### 安装步骤

```bash
# 克隆到 OpenCode skills 目录
git clone https://github.com/darkLordIceCream/Infinite-Context-Engine-SKILL.git \
  ~/.config/opencode/skills/infinite-context-engine

# 验证安装
opencode skill list | grep infinite-context-engine
```

ICE 将被 OpenCode 自动检测为可用 skill。

### 资源消耗预估

| 审议规模 | 估算 Token 消耗 | 预计耗时 | 费用参考 (DeepSeek) |
|---|---|---|---|
| 轻量审议 (3 幕) | ~30,000 tokens | ~15 分钟 | ~$0.03 |
| 标准审议 (4 幕) | ~60,000 tokens | ~30 分钟 | ~$0.06 |
| **文明量级审议 (5 幕)** | **~100,000 tokens** | **~45-60 分钟** | **~$0.10** |

> 文明量级审议的计算资源消耗，反映的是完整展开决策空间所需的审议深度。
